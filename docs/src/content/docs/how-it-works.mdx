---
title: How it works
---

Vite Deploy provides a unified set of concepts across all adapters, while still allowing each adapter to account for host-specific features and constraints.

## Adapters

Each adapter shares the same API and mental model, but its implementation differs to take advantage of the specifics of its deployment target. This means you can switch between adapters with minimal changes to your application code.

## Request handler

The request handler is the bridge between the deployment host and your application. It routes incoming HTTP requests:

- Through Vite in development and during prerendering.
- Directly to your handler in production.

## On-demand rendering (SSR)

For on-demand rendering, you provide a server entrypoint. Depending on the adapter, this can either be the same request handler as or a distinct server-only entrypoint.

## Prerendering (SSG)

For static site generation, you provide a [prerender entrypoint](/outputs/#prerenderentrypoint). This gives you explicit control over which routes are pre-rendered at build time, rather than relying on automatic route discovery.
